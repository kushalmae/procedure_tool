Five integrated tools

Each tool solves a distinct ops problem. Together, they give your team a complete operational picture.

01

Procedure Runner

Execute step-by-step checklists against your fleet. Create procedures in YAML, review before starting, and record pass/fail with notes at each step. Completed runs produce a full summary with timestamps and run notes for handover or audit.

  • Create, edit, clone, and review procedures
  • Per-satellite execution with operator tracking
  • Run notes for handover and anomaly documentation
  • Searchable history with tag filtering
View Procedures →
02

Mission Scribe

Record structured shift logs across multiple roles — Mission Director, Flight Dynamics, TNC, Payload, and Ground Systems. All entries appear in a unified mission timeline with filters and search.

  • Role-based event logging with structured metadata
  • Unified chronological mission timeline
  • Shift-based logging with handoff notes
  • Filter by role, satellite, category, and severity
Open Mission Scribe →
03

Fleet Anomaly Tracker

Capture, track, and manage anomalies across your satellite fleet. Standardized reporting with satellite, subsystem, severity, and operational impact fields. Track investigation progress from New through Resolved.

  • Structured anomaly reporting with severity classification
  • Centralized fleet-wide anomaly registry
  • Status tracking: New → Investigating → Mitigated → Resolved
  • Notes and updates with author attribution
View Anomaly Tracker →
04

Alerts & Limits Handbook

A single source of truth for operational alerts, monitoring rules, and telemetry limits. Each entry documents the parameter, warning/critical thresholds, and recommended operator response.

  • Structured alert and limit definitions
  • Warning and critical thresholds with response guidance
  • Version tracking with automatic increment on edit
  • Optional link to recovery procedures
Open Handbook →
05

FDIR Handbook

Document all Fault Detection, Isolation, and Recovery logic. Each entry describes how faults are detected onboard, what automatic responses the spacecraft executes, and which operator procedures to follow for recovery.

  • Structured FDIR definitions with fault codes
  • Onboard automated response documentation
  • Cross-links to operator recovery procedures
  • Filter by subsystem, severity, and fault type

Get started in minutes

SatOps runs locally with no external dependencies. Set up your environment and start running procedures.

1

Install dependencies

Create a virtual environment and install the requirements.

python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
2

Initialize the database

Run migrations and seed sample data for procedures, satellites, and all reference tools.

python manage.py migrate
python manage.py seed_all
3

Create a user

Create a superuser to log in and start running procedures.

python manage.py createsuperuser
4

Launch the server

Start the development server and open the app in your browser.

python manage.py runserver

Then visit http://localhost:8000 — you're ready to go.
